\ConfigurationProvider

Implementation of class gives possibility to connect the package to a shop administration configuration user interface.

The params $type and $country can be used to organize your configuration differently for different countries and paymenttypes.

Usage: Create one or more classes that implements the \ConfigurationProvider Interface (eg. one class for testing values, one for production values). The implemented functions should return the authorization values for the configuration in question.

The integration package will then call these functions to get the respective values from your database. When starting an WebPay action in your integration file, put an instance of your class as parameter to the constructor.
